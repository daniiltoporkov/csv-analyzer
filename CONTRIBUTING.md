# Contributing to CSV Analyzer

Мы придерживаемся Git Flow. Изменения вносятся через ветки:
- `feature/*` — новая функциональность
- `bugfix/*` — исправления
- `hotfix/*` — срочные исправления для релизов
- `release/*` — подготовка релиза

Перед созданием Pull Request:
1. Убедитесь, что код отформатирован (`black`, `isort`)
2. Все тесты проходят (`pytest`)
3. Добавлены новые тесты при необходимости
4. Сообщения коммитов следуют Conventional Commits

Используйте pre-commit хуки, установленные в проекте.
